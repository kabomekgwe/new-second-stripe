# Production Deployment Guide

## Overview

This guide covers deploying the Stripe Payment Application using Docker with automated CI/CD pipelines.

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    Production Environment                    │
│                                                              │
│  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐   │
│  │   Traefik    │──▶│   Frontend   │──▶│ Core Backend │   │
│  │  (SSL/Proxy) │   │   (Next.js)  │   │  (NestJS)    │   │
│  └──────────────┘   └──────────────┘   └──────┬───────┘   │
│         │                       │             │           │
│         │                       └─────────────┘           │
│         │                              │                 │
│  ┌──────┴──────────┐         ┌────────▼────────┐        │
│  │ Webhooks Backend│         │ PostgreSQL +    │        │
│  │    (NestJS)     │         │ Redis (Docker)  │        │
│  └─────────────────┘         └─────────────────┘        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## Prerequisites

- Docker Engine 24.0+
- Docker Compose v2.0+
- Domain name with DNS control
- GitHub account (for CI/CD)
- Linux server (Ubuntu 22.04 LTS recommended)

## Quick Start

### 1. Server Setup

```bash
# Install Docker
sudo apt update
sudo apt install -y docker.io docker-compose-plugin
sudo systemctl enable docker

# Initialize Docker Swarm (optional but recommended)
docker swarm init
```

### 2. Clone and Configure

```bash
git clone <your-repo-url>
cd stripe-app

# Copy and edit environment configuration
cp .env.example .env
# Edit .env with your production values
```

### 3. Setup Docker Secrets

```bash
# Set up encrypted secrets for production
./scripts/setup-docker-secrets.sh

# This will create secrets for:
# - Database credentials
# - Stripe API keys
# - Session secrets
# - Redis password
```

### 4. Deploy

```bash
# Deploy all services
./scripts/deploy.sh production

# Or for staging
./scripts/deploy.sh staging
```

## Services

### Core Backend
- **Port**: 4917
- **Health Check**: `GET /`
- **Replicas**: 2 (load balanced)
- **Resources**: 1 CPU, 1GB RAM per instance

### Webhooks Backend
- **Port**: 4923
- **Health Check**: `GET /`
- **Replicas**: 1 (single instance for webhook consistency)
- **Resources**: 0.5 CPU, 512MB RAM

### Frontend
- **Port**: 3000
- **Health Check**: `GET /`
- **Replicas**: 2
- **Resources**: 1 CPU, 1GB RAM per instance

### PostgreSQL
- **Port**: 5432 (internal only)
- **Persistent Storage**: `/var/lib/postgresql/data`
- **Backup**: Automated nightly to `./backups/`

### Redis
- **Port**: 6379 (internal only)
- **Persistence**: AOF enabled
- **Authentication**: Via Docker secret

## Configuration

### Environment Variables

Critical variables to set in `.env` or Docker secrets:

```bash
# Stripe (Required)
STRIPE_SECRET_KEY=sk_live_xxx
STRIPE_PUBLISHABLE_KEY=pk_live_xxx
STRIPE_WEBHOOK_SECRET=whsec_xxx

# Database (Auto-generated)
DB_PASSWORD=<auto-generated>

# Security (Auto-generated)
SESSION_SECRET=<auto-generated>
REDIS_PASSWORD=<auto-generated>
```

### SSL Certificates

Traefik automatically handles SSL certificates via Let's Encrypt. Update these labels in `docker-compose.prod.yml`:

```yaml
labels:
  - "traefik.http.routers.frontend.rule=Host(`yourdomain.com`)"
  - "traefik.http.routers.frontend.tls.certresolver=letsencrypt"
```

Replace `yourdomain.com` with your actual domain.

## Database Migrations

Migrations run automatically during deployment:

```bash
# Run migrations manually
sudo docker-compose -f docker-compose.prod.yml run --rm core-backend npx typeorm migration:run

# Check migration status
sudo docker-compose -f docker-compose.prod.yml exec core-backend npx typeorm migration:show
```

To add Prometheus monitoring, include these services in `docker-compose.prod.yml`:

```yaml
prometheus:
  image: prom/prometheus:latest
  volumes:
    - ./monitoring/prometheus.yml:/etc/prometheus/prometheus.yml
  ports:
    - "9090:9090"

grafana:
  image: grafana/grafana:latest
  ports:
    - "3001:3000"
  volumes:
    - grafana-data:/var/lib/grafana
```

## Backup and Restore

### Automated Backups

The `backup` service automatically backs up the database every 24 hours.

### Manual Backup

```bash
./scripts/backup-database.sh
```

### Restore from Backup

```bash
# List available backups
ls -lh ./backups/

# Complete rollback (includes DB restore + service redeploy)
./scripts/rollback.sh backup_YYYYMMDD_HHMMSS.sql.gz

# Restore only database (keeps services running)
docker exec -i postgres-prod psql -U postgres -d stripe_app < backup.sql
```

## CI/CD Pipeline

### GitHub Actions

The repository includes automated CI/CD:

- **CI** (`.github/workflows/ci.yml`): Runs on every PR
  - Linting
  - Unit tests with coverage
  - Docker build tests
  - Security scanning

- **Deploy** (`.github/workflows/deploy.yml`): Runs on merge to main
  - Builds and pushes images to GitHub Container Registry
  - Automated deployment to production
  - Health checks
  - Slack notifications

### Setting Up Stripe Webhooks

```bash
# Run the setup script on your server
./scripts/setup-stripe-webhooks.sh

# This will:
# - Install stripe-cli if needed
# - Configure webhook endpoints in Stripe Dashboard
# - Generate webhook secret (save it to Docker secrets!)
```

## Troubleshooting

### Service Won't Start

```bash
# Check logs
sudo docker-compose -f docker-compose.prod.yml logs [service-name]

# Check container status
sudo docker ps -a

# Restart specific service
sudo docker-compose -f docker-compose.prod.yml restart core-backend
```

### Database Connection Issues

```bash
# Check if PostgreSQL is running
sudo docker-compose -f docker-compose.prod.yml exec postgres pg_isready

# Check Redis
sudo docker-compose -f docker-compose.prod.yml exec redis redis-cli ping
```

### Webhook Failures

1. Check webhooks service: `sudo docker-compose -f docker-compose.prod.yml logs webhooks-backend`
2. Verify Stripe webhook secret: `sudo docker secret ls`
3. Check Stripe Dashboard for webhook delivery attempts

### SSL Certificate Issues

```bash
# Check Traefik logs
sudo docker-compose -f docker-compose.prod.yml logs traefik

# Force certificate regeneration (remove and restart)
sudo docker volume rm stripe-app_traefik-certs
sudo docker-compose -f docker-compose.prod.yml restart traefik
```

## Security

### Docker Secrets

All sensitive data is stored in Docker secrets, not environment variables:

```bash
# List secrets
sudo docker secret ls

# Rotate a secret
sudo docker secret rm stripe_secret_key
sudo docker secret create stripe_secret_key /path/to/new_key.txt

# Restart service to pickup new secret
sudo docker-compose -f docker-compose.prod.yml restart core-backend webhooks-backend
```

### Security Headers

The application includes security headers via Helmet:

- Content Security Policy
- HSTS
- X-Frame-Options
- X-Content-Type-Options

### Rate Limiting

- Core API: 100 requests/minute (adjust in `core-backend/src/app.module.ts`)
- Auth endpoints: 5 requests/minute per endpoint
- Webhooks: 500 requests/minute for Stripe event handling

## Scaling

### Horizontal Scaling

```bash
# Scale core backend to 3 instances
sudo docker-compose -f docker-compose.prod.yml up -d --scale core-backend=3 core-backend

# Frontend is already stateless and can be scaled
sudo docker-compose -f docker-compose.prod.yml up -d --scale frontend=4 frontend

# Note: Webhooks should remain at 1 instance (no horizontal scaling)
```

### Vertical Scaling

Edit resource limits in `docker-compose.prod.yml`:

```yaml
deploy:
  resources:
    limits:
      cpus: '2.0'  # Increase from 1.0
      memory: 2G   # Increase from 1G
```

## Maintenance

### System Updates

```bash
# Update Docker images
sudo docker-compose -f docker-compose.prod.yml pull
sudo docker-compose -f docker-compose.prod.yml up -d

# Clean up old images
sudo docker system prune -f
```

### Log Rotation

Docker handles log rotation automatically. To configure:

```bash
# Edit daemon.json
sudo nano /etc/docker/daemon.json

# Add:
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}

# Restart Docker
sudo systemctl restart docker
```
